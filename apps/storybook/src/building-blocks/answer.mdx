# Answer

Answer used as an option in Radio, Checkbox or Select

## Properties

<Table>
  {common.thead}
  <tbody>
    <tr>
      <td>
        `type`
      </td>
      <td>
        The type is required in order to determine what type of node it is. For
        an answer, this property must be set to `Answer`.
      </td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>
        `id`
      </td>
      <td>The id of the answer input</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>
        `heading`
      </td>
      <td>The title for the option.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>
        `value`
      </td>
      <td>
        Option value. What is set in the data object, not visible to the user.
      </td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>
        `text`
      </td>
      <td>Additional description that follows the heading for the answer.</td>
      <td></td>
    </tr>
    <tr>
      <td>
        `image`
      </td>
      <td>
        Image to show with description. If a values is passed, the following
        structure is expected:
```ts
{
  url: 'https://url.to/image.jpg',
  alt: 'alternative text'
}
```
      </td>
      <td></td>
    </tr>
    <tr>
      <td>
        `messages`
      </td>
      <td>
        Array of `Message` nodes that can show on an Answer. The first message with either 
        no test, or a test that yields a truthy value will be returned. No more than one
        message will be rendered.
      </td>
      <td></td>
    </tr>
    {common.show}
    {common.hide}
  </tbody>
</Table>
